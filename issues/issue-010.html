Claude Code vs OpenCode Token 开销实测：Claude Code 在收到提示前已发送 33K token
Systima.ai 发布编码智能体 token 开销深度对比：最小任务（回复"OK"）Claude Code 冷启动发送 ~32,800 token，OpenCode 仅 ~6,900 token（4.7x）。差距主要源于 Claude Code 的 27 个工具定义（~24K token）vs OpenCode 的 10 个。更大瓶颈在缓存：Claude Code 因不稳定前缀在同会话中多次重写数万 token，缓存写入达到 OpenCode 的 5.9–54 倍。结论：账单差异来自工具架构，而非模型质量。

迁移实践
Ploy 详述 GPT-5.6 Sol 生产迁移：2.2x 速度提升，27% 成本降低，但缓存架构需彻底重建
AI 建站平台 Ploy 发布详尽迁移指南，将默认模型从 Claude Opus 4.8 切换至 GPT-5.6 Sol。实测成绩：构建完成 2.2x 加速（8 分钟→3 分 42 秒）、$3.06→$2.22 单构建成本、视觉评分 0.936→0.970。但迁移过程暴露了三个关键工程问题——GPT-5.6 强制填满所有可选参数（需 schema 改写为 nullable 解决空读）、OpenAI 放弃隐式前缀缓存导致首次缓存命中率接近 0%（需 workspace 级键空间设计）、reasoning replay 需自包含化。文章被 HN 推至首页（211pts），成为 GPT-5.6 迁移的事实参考指南。
